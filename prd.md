## 一、專業考題

### Gen AI 自動回覆平台後端系統

#### 背景說明

MaiAgent 專注於生成式 AI 平台，產品應用範圍涵蓋客服與知識管理（KM）等場景。系統核心在於透過 AI 自動回覆使用者的查詢，並且根據不同的場景（例如客服詢問或內部知識查詢）使用不同的預設設定與處理邏輯。

#### 任務目標

請你從 [cookiecutter-django](https://github.com/cookiecutter/cookiecutter-django) 這個專案範本產生一個新的 Django 專案，並基於此專案設計與實作以下功能。重點在於展示如何針對具體業務邏輯進行架構規劃、資料模型設計、非同步處理（例如呼叫 AI 生成 API）、以及 API 規劃。

#### 功能需求（業務邏輯面向）

**對話管理：**

- 設計一個 model 來儲存用戶的會話歷史，每筆會話應包含使用者資訊、會話狀態、時間戳記、對話內容。

**AI 自動回覆流程：**

- 當使用者送出訊息時，系統應接收並先記錄查詢內容，然後透過非同步任務模擬呼叫外部生成式 AI API 來生成回覆。
- 回覆結果回傳後，要更新對話記錄，同時提供一個簡單的 RESTful API 讓前端取得回覆結果。

**API 與管理介面：**

- 實作一組 API endpoint，用於提交用戶查詢、查詢會話紀錄與更新場景設定。API 設計要考慮安全性與效能。
- 可以利用 Django Admin 或自訂介面，讓業務人員能夠檢視與管理歷史對話記錄，並進行必要的監控與調整。

**擴充性：**

- 設計時考慮可擴充性，例如未來可能要接入更多 AI 模型、更複雜的路由邏輯等。

#### 實作說明

- **初始環境：** 須從 cookiecutter-django 產生新的專案，並以此為基礎進行擴充。專案架構與設定可參考範本提供的最佳實踐，但針對我們的業務邏輯要做必要的調整。
- **業務邏輯重點：** 強調資料模型設計、非同步任務流程（如 Celery 任務設計）、以及 API 的整合。重點不在於 UI 呈現，而是能否清楚規劃整體系統流程，並將各個模組有機結合。
- **開發過程：** 想像這是一個多人協作的場景，你透過 PR, PR Review 來完成此任務。
- **測試與文件（非必要）：** 請簡述主要設計考量，並用簡要的文件說明各模組之間的交互關係。也鼓勵撰寫基本的單元測試來驗證主要邏輯。

#### 評分重點

- **架構規劃：** 能否從 cookiecutter-django 的基礎上，根據需求規劃出合理的專案結構與模組劃分。
- **資料模型設計：** 是否能設計出清晰且具彈性的資料模型，滿足不同場景的需求。
- **業務邏輯實現：** 非同步任務、API 設計與錯誤處理等實作能否切合實際業務場景，並兼顧效能與安全性。
- **可擴充性與維護性：** 是否考慮到未來擴充需求、程式碼的可讀性。

#### 附加挑戰（非必選）

- **進階搜尋功能：** 為對話記錄提供全文檢索功能，讓管理人員能快速查詢特定關鍵字。
- **多 AI 模型支持：** 設計一個機制，根據不同場景自動選擇不同的 AI 模型或回覆模板，並讓管理人員可以動態調整權重與路由邏輯。
